### Findings Detail:

**FINDING-001 (HIGH) — Stored XSS**
- API accepts `<script>alert(1)</script>` as agent name and stores raw in DB
- Response echoes the raw XSS string in success message
- React JSX escapes it on render (no execution in current UI) but data layer is unsafe
- If any future code uses `innerHTML`, it becomes exploitable

**FINDING-002 (MEDIUM) — No input validation on IDs**
- Skill ID `'; DROP TABLE skills;--` accepted and stored as-is
- Not exploitable today (parameterized queries) but dangerous pattern

**Recommendations:**
1. Sanitize/strip HTML tags from all string inputs server-side
2. Add regex validation on ID fields (alphanumeric + hyphens only)
3. Normalize error messages (JSON parse errors leak position info)
